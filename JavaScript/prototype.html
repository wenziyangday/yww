原型和原型链 ​

原型 ​

js规定每一个函数都有一个prototype（原型对象）[原型对象] 每一个实例对象都有一个__proto__（私有属性）[实例对象]

原型链 ​

原型链是 JavaScript 中实现对象继承的一种机制，它基于原型（prototype）和链式结构。
每个 JavaScript 对象都有一个原型对象，而原型对象又可以有自己的原型，形成一条原型链。

工作原理 ​

  1. 每个对象都有一个原型对象
    • 在 JavaScript 中，几乎所有的对象都是通过构造函数创建的，而构造函数又有一个 prototype 属性，指向该构造函数的原型对象。
    • 当使用构造函数创建一个新对象时，该对象会自动拥有构造函数的原型对象作为自己的原型。
  2. 对象之间通过原型连接
    • 如果一个对象 A 需要访问一个属性或方法，而该属性或方法不在对象 A 自身的属性中，JavaScript 引擎会自动到对象 A 的原型对象中查找，
    • 如果还没有找到，就会继续向上查找原型对象的原型对象，直到找到为止。
    • 这样就形成了一条原型链。
  3. 原型链的终点是 Object.prototype
    • 所有对象的原型链的最终都会指向 Object.prototype，它是所有 JavaScript 对象的根原型。
    • 如果在整条原型链上都没有找到需要的属性或方法，那么 JavaScript 引擎会返回 undefined。
  4. 原型链实现继承
    • 通过原型链，子对象可以继承父对象的属性和方法，即子对象的原型对象是父对象。
    • 这种继承方式称为原型继承，它是 JavaScript 中实现对象继承的基础。

总之，原型链实现了 JavaScript 中的对象继承和属性查找机制，使得对象之间可以共享属性和方法，并且可以形成一种层级结构，方便了属性和方法的继承和访问。

对象继承 ​

  1. class 类型 extends

  2. 构造函数类型：

    • 原型链继承（原型式继承）

      原型链继承，也称为原型式继承，是一种利用原型链的机制实现继承的方式。它的核心思想是利用已有对象作为新对象的原型 ，从而实现对其属性和方法的继承。

      js
        // 父类（原型对象）
  function Animal(name) {
   this.name = name;
  }
    
  // 父类的原型方法
  Animal.prototype.sayName = function() {
   console.log('My name is ' + this.name);
  }
    
  // 子类（继承自父类）
  function Dog(name, age) {
   this.age = age;
  }
    
  // 将父类的实例作为子类的原型
  Dog.prototype = new Animal();
    
  // 子类的原型方法
  Dog.prototype.sayAge = function() {
   console.log('My age is ' + this.age);
  }
    
  // 创建子类实例
  var dog = new Dog('Buddy', 2);
    
  // 调用继承的方法
  dog.sayName(); // 输出：My name is Buddy
  dog.sayAge(); // 输出：My age is 2

      在上面的示例中，Animal 是父类，Dog 是子类，通过将 Dog.prototype 设置为一个 Animal 的实例，实现了子类对父类属性和方法的继承。这样一来，Dog 的实例就可以调用父类 Animal 的 sayName 方法了。

    • 构造函数继承

      构造函数继承是一种通过在子类构造函数中调用父类构造函数来实现继承的方式。这种方式可以实现属性的继承，但无法继承父类原型上的方法。

      js
        // 父类（超类）
  function Animal(name) {
    this.name = name;
  }
    
  // 父类的实例方法
  Animal.prototype.sayName = function() {
    console.log('My name is ' + this.name);
  }
    
  // 子类（子类构造函数内部调用父类构造函数实现继承）
  function Dog(name, age) {
    // 继承父类的属性
    Animal.call(this, name);
    // 子类自己的属性
    this.age = age;
  }
    
  // 创建子类实例
  var dog = new Dog('Buddy', 2);
    
  // 子类实例可以访问父类的属性
  console.log(dog.name); // 输出：Buddy
  // 但是无法访问父类原型上的方法
  dog.sayName(); // 报错：dog.sayName is not a function

      在上面的示例中，Animal 是父类，Dog 是子类，通过在子类构造函数内部调用父类构造函数 Animal.call(this, name) 来实现对父类属性的继承。这样一来，Dog 的实例就可以访问到父类的属性了。但是由于并没有将父类的原型赋值给子类，所以无法继承父类原型上的方法。

    • 组合继承

      组合继承是一种继承方式，结合了原型链继承和构造函数继承的优点，实现了属性和方法的完全继承。

      js
       // 父类（超类）
 function Animal(name) {
   this.name = name;
 }

 // 父类的实例方法
 Animal.prototype.sayName = function() {
   console.log('My name is ' + this.name);
 }

 // 子类（子类构造函数内部调用父类构造函数实现继承）
 function Dog(name, age) {
    // 继承父类的属性
    Animal.call(this, name);
    // 子类自己的属性
    this.age = age;
 }

 // 子类继承父类的方法
 Dog.prototype = new Animal();

 // 修正子类的构造函数指向
 Dog.prototype.constructor = Dog;

 // 创建子类实例
 var dog = new Dog('Buddy', 2);

 // 子类实例可以访问父类的属性和方法
 console.log(dog.name); // 输出：Buddy
 dog.sayName(); // 输出：My name is Buddy

      在上面的示例中，我们通过将子类的原型指向父类的实例来实现对父类方法的继承，然后通过修改子类原型的构造函数指向来修正子类的构造函数。这样一来，子类就同时继承了父类的属性和方法。

    • 实例继承

      实例继承是一种通过复制现有对象的属性和方法来创建新对象的继承方式。在 JavaScript 中，可以使用 Object.create() 方法来实现实例继承。这个方法会创建一个新对象，并将现有对象作为新对象的原型。

      js
        // 现有对象（父对象）
  var parent = {
    name: 'Parent',
    sayHello: function() {
      console.log('Hello from ' + this.name);
    }
  };
  
  // 创建子对象，并将父对象作为其原型
  var child = Object.create(parent);
  
  // 子对象继承了父对象的属性和方法
  console.log(child.name); // 输出：Parent
  child.sayHello(); // 输出：Hello from Parent

      在上面的示例中，parent 对象是现有的父对象，child 对象是通过 Object.create(parent) 创建的新对象，它继承了 parent 对象的属性和方法。

    • 拷贝继承

      拷贝继承是一种通过复制现有对象的属性和方法来创建新对象的继承方式。 在 JavaScript 中，可以通过遍历对象的属性并将其复制到新对象来实现拷贝继承。

      js
                    // 现有对象（父对象）
  var parent = {
      name: 'Parent',
      sayHello: function() {
          console.log('Hello from ' + this.name);
      }
  };
  
  // 创建一个空对象作为子对象
  var child = {};
  
  // 将父对象的属性和方法复制到子对象中
  for (var key in parent) {
      if (parent.hasOwnProperty(key)) { // 确保只复制父对象自身的属性，而不是继承的属性
          child[key] = parent[key];
      }
  }
  
  // 子对象继承了父对象的属性和方法
  console.log(child.name); // 输出：Parent
  child.sayHello(); // 输出：Hello from Parent

      在上面的示例中，parent 对象是现有的父对象，child 对象是通过遍历 parent 对象的属性并将其复制到一个新对象中来创建的。这样，child 对象就继承了 parent 对象的属性和方法。

    • 寄生继承

      寄生继承是一种继承方式，它与原型链继承、构造函数继承、组合继承等类似，通过创建新对象来继承现有对象的属性和方法。寄生继承的特点是在继承过程中增强或扩展新对象。

      js
        // 现有对象（父对象）
  var parent = {
      name: 'Parent',
      sayHello: function() {
          console.log('Hello from ' + this.name);
      }
  };
  
  // 寄生继承函数
  function inheritObject(o) {
      // 创建一个空对象作为子对象
      var child = Object.create(o);
  
      // 增强子对象，添加新方法
      child.sayGoodbye = function() {
          console.log('Goodbye from ' + this.name);
      };
  
      return child; // 返回增强后的子对象
  }
  
  // 创建子对象并继承父对象的属性和方法
  var child = inheritObject(parent);
  
  // 子对象继承了父对象的属性和方法，并且增强了新方法
  console.log(child.name); // 输出：Parent
  child.sayHello(); // 输出：Hello from Parent
  child.sayGoodbye(); // 输出：Goodbye from Parent

      在上面的示例中，inheritObject 函数接受一个现有对象 o 作为参数，然后通过 Object.create(o) 创建了一个继承自 o 的空对象 child。在 inheritObject 函数中，还向 child 对象添加了一个新的方法 sayGoodbye。这样，通过寄生继承，我们实现了子对象继承父对象的属性和方法，并且增强了新方法。

  3. 优缺点

    1. 原型链继承：
      • 优点：
        • 实现简单，易于理解。
        • 可以继承原型链上的属性和方法。
      • 缺点：
        • 所有子对象共享同一个原型对象，一个子对象修改原型对象可能会影响其他子对象。
        • 不能向父类构造函数传递参数。
    2. 构造函数继承：
      • 优点：
        • 避免了原型链继承的共享问题，每个子对象都有自己的构造函数。
        • 可以向父类构造函数传递参数。
      • 缺点：
        • 不能继承原型链上的属性和方法，只能继承构造函数内部定义的属性和方法。
        • 每个子对象都会单独拷贝一份父对象的属性和方法，导致内存浪费。
    3. 组合继承：
      • 优点：
        • 兼具原型链继承和构造函数继承的优点，解决了它们各自的缺点。
        • 可以继承原型链上的属性和方法，并且可以向父类构造函数传递参数。
      • 缺点：
        • 调用两次父类构造函数，造成子对象原型上的属性和方法重复。
    4. 实例继承：
      • 优点：
        • 简单易用，不需要使用 new 关键字，直接通过 Object.create() 创建子对象。
      • 缺点：
        • 只能继承父对象的属性，无法继承父对象的方法。
        • 每个子对象都会单独拷贝一份父对象的属性，导致内存浪费。
    5. 拷贝继承：
      • 优点：
        • 简单易用，可以直接将父对象的属性和方法拷贝给子对象。
      • 缺点：
        • 每个子对象都会单独拷贝一份父对象的属性和方法，导致内存浪费。
        • 无法继承父对象的原型链上的属性和方法。
    6. 寄生继承：
      • 优点：
        • 可以实现类似于构造函数继承的效果，但是避免了直接修改父对象的缺点。
      • 缺点：
        • 与构造函数继承一样，不能继承父对象的原型链上的属性和方法。

    综合来看，组合继承在实际开发中应用较为广泛，因为它兼具了原型链继承和构造函数继承的优点，并且解决了它们的缺点。
