keyof & typeof ​

keyof ​

md
keyof运算符 采用对象类型，并生成其键的字符串或数字文字 并集（联合类型）

基础数据类型使用keyof会出现隐式转化

ts
type TNumber = keyof 1;
// 等价于
type TNumberEqualType = keyof Number;
type TNumberEqual = "toString" | "toFixed" | "toExponential" | "toPrecision" | "valueOf" | "toLocaleString"

type TString = keyof '1';
// 等价于
type TStringEqualType = keyof String;
type TStringEqual =
  number
  | "toString"
  | "charAt"
  | "charCodeAt"
  | "concat"
  | "indexOf"
  | "lastIndexOf"
  | "localeCompare"
  | "match"
  | "replace"
  | "search"
  | '等等多个interface属性'

type TBoolean = keyof false;
// 等价于
type TBooleanEqualType = keyof Boolean;
type TBooleanEqual = 'valueOf'

Number、String、Boolean 的keyof 来源于 lib.es5.d.ts 的 interface Number、interface String 、interface Boolean

ts
type Arrayish = { [n: number]: unknown };
type A = keyof Arrayish;
// 等价于
type AEqual = number

type Mapish = { [k: string]: boolean };
type M = keyof Mapish;
// 等价于
type MEqual = string | number

M 的类型 string | number， 这是因为JavaScript对象键总是被强制转化为字符串，所以obj[0]总是与obj[“0”]相同

typeof ​

markdown
javascript 中可以通过typeof获取数据的类型，
而在typescript中使用typeof 可以为变量添加类型注解
ts
let name = '张三';
let firstName: typeof name = '张';
// 等价于
let firstNameEqual: string = '张';

function f() {
  return {
    x: 1,
    y: 2
  }
}

// TS2749: 'f' refers to a value, but is being used as a type here. Did you mean 'typeof f'?
type P = ReturnType<f>

type Tp = ReturnType<typeof f>
// 等价于
type TpEqual = {
  x: number,
  y: number
}

区别 ​

javascript 中对函数 使用typeof得到的是一个字符串function，而typescript 中对于函数使用typeof则是 对函数返回值做了一次类型推导 返回值是对象的则是对象中每个属性的类型推导（隐式做了一次type） 返回值是字符串、数字、false 则对应的应该是string、number、boolean 没有返回值的则是void 这是函数默认的类型推导

tips: 只有在标志符（函数的标志符是函数的名称）或者属性上使用typeof 才是合法的

ts
// Meant to use = ReturnType<typeof msgbox>
// 下面是不合法的 会出现报错
let shouldContinue: typeof msgbox("Are you sure you want to continue?");
// ',' expected.
