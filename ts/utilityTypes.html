工具类 Utility Types ​

md
Typescript 内置工具类，用于常见的类型转换

Awaited<Type> ​

这种类型旨在模拟 async 函数中的 await 之类的操作，或 Promise 上的 .then() 方法 - 特别是它们递归解包 Promise 的方式，直至返回类型中不再出现Promise类型

ts
type A = Awaited<Promise<string>>;            // type A = string
type B = Awaited<Promise<Promise<number>>>;   // type B = number
type C = Awaited<boolean | Promise<number>>;  // type C = number | boolean

Partial<Type> ​

构造一个将 Type 的 所有属性 设置为可选的类型。 此实用程序将返回一个表示给定类型的所有子集的类型。

ts
type Parti = {
  name: string,
  age: number,
  sex: string
}

const parti: Parti = {
  name: '张胜男',
  age: 100,
  sex: '那么'
}

const parti2: Partial<Parti> = {}

Required<Type> ​

构造一个由设置为 required 的 Type 的 所有属性 组成的类型。 与 Partial 相反。

ts
type Parti = {
  name: string,
  age: number,
  sex: string
}

const parti: Parti = {
  name: '张胜男',
  age: 100,
  sex: '那么'
}

type Req = Required<Partial<Parti>>
const req: Req = {
  name: '张胜男',
  age: 100,
  sex: '那么'
}

Readonly<Type> ​

构造一个将 Type 的 所有属性 设置为 readonly 的类型，这意味着构造类型的属性不能重新分配( 无法进行重新赋值，否则会报错)。

ts
interface Todo {
  title: string;
}

const todo: Readonly<Todo> = {
  title: "Delete inactive users",
};

// TS2540: Cannot assign to 'title' because it is a read-only property.
todo.title = "Hello";

Object.freeze() ​

Object.freeze() 静态方法可以使一个对象被冻结。 冻结对象可以防止扩展，并使现有的属性不可写入和不可配置。 被冻结的对象不能再被更改：不能添加新的属性，不能移除现有的属性，不能更改它们的可枚举性、可配置性、可写性或值，对象的原型也不能被重新指定。 freeze() 返回与传入的对象相同的对象。 冻结一个对象是 JavaScript 提供的最高完整性级别保护措施。

ts
function freeze<Type>(obj: Type): Readonly<Type>;

function freeze() {

}

Record<Keys, Type> ​

构造一个对象类型，其 属性键 为 Keys，其 属性值 为 Type。 此实用程序可用于将一种类型的属性映射到另一种类型。

ts
interface CatInfo {
  age: number;
  breed: string;
}

type CatName = "miffy" | "boris" | "mordred";

const cats: Record<CatName, CatInfo> = {
  miffy: {age: 10, breed: "Persian"},
  boris: {age: 5, breed: "Maine Coon"},
  mordred: {age: 16, breed: "British Shorthair"},
};

cats.boris;

const cats2: Record<string, CatInfo> = {
  dog: {age: 10, breed: "Persian"},
}

cats2.dog;

Pick<Type, Keys> ​

从Type 中 挑选 一部分属性 Keys （字符串字面量 或 字符串字面量的联合类型 Union types）来构造一个类型 检出需要的

ts
interface Todo {
  title: string;
  description: string;
  completed: boolean;
}

type TodoPreview = Pick<Todo, "title" | "completed">;

const todo: TodoPreview = {
  title: "Clean room",
  completed: false,
};

todo;

Omit<Type, Keys> ​

从 Type 中进行 Keys 的 缺省（移除） , 缺省后再添加会报错。 省略不需要的（移除不需要的）

ts
interface Todo {
  title: string;
  description: string;
  completed: boolean;
  createdAt: number;
}

type TodoPreview = Omit<Todo, "description">;

const todo: TodoPreview = {
  title: "Clean room",
  completed: false,
  createdAt: 1615544252770,
};

todo;

type TodoInfo = Omit<Todo, "completed" | "createdAt">;

const todoInfo: TodoInfo = {
  title: "Pick up kids",
  description: "Kindergarten closes at 5pm",
};

todoInfo;

区别于可选属性： 可选属性是对象中有无，不会产生编译问题，而Omit构造的类型，创建对象重新填入，会报错

Exclude<UnionType, ExcludedMembers> ​

从 所有的联合类型成员（UnionType）中 排除 指定的类型(ExcludedMembers), 从而构造一个类型。 获取 类型差集

ts
type T0 = Exclude<"a" | "b" | "c", "a">;
type T0Equal = "b" | "c"

type T1 = Exclude<"a" | "b" | "c", "a" | "b">;
type T1Equal = "c"

type T2 = Exclude<string | number | (() => void), Function>;
type T2Equal = string | number

type Shape =
  | { kind: "circle"; radius: number }
  | { kind: "square"; x: number }
  | { kind: "triangle"; x: number; y: number };
type T3 = Exclude<Shape, { kind: "circle" }>
type T3Equal = {
  kind: "square";
  x: number;
} | {
  kind: "triangle";
  x: number;
  y: number;
}

Extract<Type, Union> ​

从 所有类型成员Type 中，提取可以分配给 联合类型Union的成员，从而构造一个类型。 获取类型交集

ts
type T0 = Extract<"a" | "b" | "c", "a" | "f">;
type T0Equal = "a"

type T1 = Extract<string | number | (() => void), Function>;
type T1Equal = () => void

type Shape =
  | { kind: "circle"; radius: number }
  | { kind: "square"; x: number }
  | { kind: "triangle"; x: number; y: number };

type T2 = Extract<Shape, { kind: "circle" }>
type T2Equal = {
  kind: "circle";
  radius: number;
}

NonNullable<Type> ​

构造一个不存在undefined或者null的类型

ts
type T0 = NonNullable<string | number | undefined>;
// 等价于
type T0Equal = string | number

type T1 = NonNullable<string[] | null | undefined>;
// 等价于
type T1Equal = string[]

Parameters<Type> ​

从函数类型 Type 的 参数 中，使用 类型构造 构造出一个元组类型

ts
declare function f1(arg: { a: number; b: string }): void;

type T0 = Parameters<() => string>;
type T0Equal = []

type T1 = Parameters<(s: string) => void>;
type T1Equal = [s: string]

type T2 = Parameters<<T>(arg: T) => T>;
type T2Equal = [arg: unknown]

type T3 = Parameters<typeof f1>;
type T3Equal = [arg: {
  a: number;
  b: string;
}]

type T4 = Parameters<any>;
type T4Equal = unknown[]

type T5 = Parameters<never>;
type T5Equal = never

type T6 = Parameters<string>;
// Type 'string' does not satisfy the constraint '(...args: any) => any'.
type T6Equal = never

type T7 = Parameters<Function>;
// Type 'Function' does not satisfy the constraint '(...args: any) => any'.
//   Type 'Function' provides no match for the signature '(...args: any): any'.
type T7Equal = never

ConstructorParameters<Type> ​

从 构造函数 类型 Type 的 参数 中，使用 类型构造 构造出一个元组类型或者数组类型。 它生成一个包含所有参数类型的元组类型（如果 Type 不是函数，则生成类型 never）。

ts
type T0 = ConstructorParameters<ErrorConstructor>;
type T0Equal = [message?: string]

type T1 = ConstructorParameters<FunctionConstructor>;
type T1Equal = string[]

type T2 = ConstructorParameters<RegExpConstructor>;
type T2Equal = [pattern: string | RegExp, flags?: string]

class C {
  constructor(a: number, b: string) {
  }
}

type T3 = ConstructorParameters<typeof C>;
type T3Equal = [a: number, b: string]

type T4 = ConstructorParameters<any>;
type T4Equal = unknown[]

type T5 = ConstructorParameters<Function>;
// Type 'Function' does not satisfy the constraint 'abstract new (...args: any) => any'.
//   Type 'Function' provides no match for the signature 'new (...args: any): any'.
type T5Equal = never

ReturnType<Type> ​

构造一个由函数 Type 的 返回类型 组成的类型。

ts

declare function f1(): { a: number; b: string };

type T0 = ReturnType<() => string>;
type T0Equal = string

type T1 = ReturnType<(s: string) => void>;
type T1Equal = void

type T2 = ReturnType<<T>() => T>;
type T2Equal = unknown

type T3 = ReturnType<<T extends U, U extends number[]>() => T>;
type T3Equal = number[]

type T4 = ReturnType<typeof f1>;
type T4Equal = {
  a: number;
  b: string;
}


type T5 = ReturnType<any>;
type T5Equal = any

type T6 = ReturnType<never>;
type T6Equal = never

type T7 = ReturnType<string>;
// Type 'string' does not satisfy the constraint '(...args: any) => any'.
type T7Equal = any


type T8 = ReturnType<Function>;
// Type 'Function' does not satisfy the constraint '(...args: any) => any'.
// Type 'Function' provides no match for the signature '(...args: any): any'.
type T8Equal = any

对于重载函数，则返回最后一个函数签名的 返回值类型

InstanceType<Type> ​

构造一个由 Type 中的构造函数的 实例类型 组成的类型。

ts
class C {
  x = 0;
  y = 0;
}
type T0 = InstanceType<typeof C>;
// type T0 = C

type T1 = InstanceType<any>;
// type T1 = any

type T2 = InstanceType<never>;
// type T2 = never

type T3 = InstanceType<string>;
// Type 'string' does not satisfy the constraint 'abstract new (...args: any) => any'.
// type T3 = any

type T4 = InstanceType<Function>;
// Type 'Function' does not satisfy the constraint 'abstract new (...args: any) => any'.
// Type 'Function' provides no match for the signature 'new (...args: any): any'.
// type T4 = any

ThisParameterType<Type> ​

提取函数类型的 this 参数 的类型，如果函数类型没有 this 参数，则提取 unknown。

ts
function toHex(this: Number) {
  return this.toString(16);
}

function numberToString(n: ThisParameterType<typeof toHex>) {
  return toHex.apply(n);
}

OmitThisParameter<Type> ​

从 Type 中 删除 this 参数。 如果 Type 没有显式声明的 this 参数，则结果只是 Type。 否则，将从 Type 创建一个没有 this 参数的新函数类型。 泛型被删除，只有最后一个重载签名被传播到新的函数类型中。

ts
function toHex(this: Number) {
  return this.toString(16);
}

const fiveToHex: OmitThisParameter<typeof toHex> = toHex.bind(5);
console.log(fiveToHex());

ThisType<Type> ​

此实用程序不返回转换后的类型。 相反，它用作上下文 this 类型的标记。 请注意，必须启用 noImplicitThis 标志才能使用此实用程序。

ts
type ObjectDescriptor<D, M> = {
  data?: D;
  methods?: M & ThisType<D & M>; // Type of 'this' in methods is D & M
};

function makeObject<D, M>(desc: ObjectDescriptor<D, M>): D & M {
  let data: object = desc.data || {};
  let methods: object = desc.methods || {};
  return {...data, ...methods} as D & M;
}

let obj = makeObject({
  data: {x: 0, y: 0},
  methods: {
    moveBy(dx: number, dy: number) {
      this.x += dx; // Strongly typed this
      this.y += dy; // Strongly typed this
    },
  },
});

obj.x = 10;
obj.y = 20;
obj.moveBy(5, 5);

在上面的示例中，makeObject 参数中的 methods 对象具有包含 ThisType<D & M> 的上下文类型， 因此 methods 对象内的方法中的 this类型是 { x: number, y: number } & { moveBy(dx: number, dy: number): void }。 请注意 methods 属性的类型如何同时是方法中 this 类型的推断目标和源。 ThisType<T> 标记接口只是在 lib.d.ts 中声明的一个空接口。 除了在对象字面的上下文类型中被识别之外，接口的行为就像任何空接口。
